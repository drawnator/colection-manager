  1. E se quiséssemos garantir que os objectos req e res fossem sempre utilizados da forma correta, sem depender só da nossa memória ou sorte? com base no que discutimos sobre TypeScript, como ele poderia nos ajudar nesse cenário?
  2. Como você imaginaria esse mesmo código escrito em TypeScript? O que mudaria na estrutura do código? Quais benefícios você esperaria ter? E quais possíveis dificuldades surgiriam?

Respostas:

  1. O TypeScript serviria para verificar a tipagem antes da geração do codigo .js já que um compila no outro e permitira com que definissemos tipos e como podemos ou não podemos interagir com ele
  2. Imagino que seria algo como:
    const express:Express = require('express');
    const path:Path = require('path');
    const app:App = express();
    const port:int = 3000;

    app.use(express.static(path.join(__dirname,'public')));

    app.get('/',(req,res) =>{
    res.end('Hello, World!\n');
    });

    app.listen(port, () => {
    console.log(`server running at http://localhost:${port}/`);
    });

     Os beneficios para um código simples são são tão aparentes mas as vantagens devem aparecer com o amento da complexidade do projeto dado a segurança adicional provida pela tipagem estática no projeto.
    Existe a possibilidade dessa falta de flexibilidade faça o processo de desenvolvimento ficar mais lento imagino que essa seria a principal dificuldade acrescentada
